# Personal blog

Personal website with articles about software design, development, and experiments.

The project is intentionally minimal: everything is built using tools I already
use daily from the terminal, keeping the writing and publishing workflow simple
and distraction-free.

---

## The `PACH` Stack

This blog is built using what I like to call the **PACH stack**:

- **PA**ndoc → Markdown → HTML conversion  
- **C**SS → styling  
- **H**TML → final output  

No JavaScript, no frameworks, no runtime dependencies.

Make is used as a **build orchestrator**.

## Build workflow

The site is entirely static.

### Articles

- Each .md file in ./md is converted into an HTML page using pandoc;
- A custom template (templates/article.html) is applied;
- Output is written to ./docs/articles/;

### RSS feed

- The RSS feed is generated by a small Python script;
- It reads metadata from Markdown files;
- Converts content to HTML using pandoc;
- Produces docs/feed.xml;

Each article is fully embedded in the feed, so readers don’t need to open the site.

## Usage

Build everything:

```bash
make
```

Use `make clean` to remove all the generated files and `make list` to show the list of the generated articles.

## Philosophy

- Write in Markdown;
- Use simple, composable CLI tools;
- Avoid unnecessary dependencies;
- Keep the build process transparent and hackable;
